program test_energies
 

Detailed Description

Validation tests for the energies module.

Overview

This program performs independent unit tests of the three energy functions provided by the energies module:

  • Kinetic energy \(T\)
  • Lennard-Jones potential energy \(V\)
  • Total energy \(E = T + V\)

Each function is tested in isolation using analytically trivial configurations, allowing direct verification of correctness without numerical ambiguity.

Test 1: Kinetic energy "T"

The kinetic energy is defined as:

\[ T = \frac{1}{2} \sum_i m_i v_i^2 \]

Test setup

  • Two atoms are defined, but only one carries velocity.
  • Mass of atom 1: \(m = 2\)
  • Velocity of atom 1: \(v = 1\)
  • All other velocities are zero.

Expected result

\[ T = \frac{1}{2} \cdot 2 \cdot 1^2 = 1.0 \]

The test verifies that the function T returns this exact value within a numerical tolerance of \(10^{-6}\).

Test 2: Lennard-Jones potential energy "V"

The Lennard-Jones potential is given by:

\[ V(r) = 4 \epsilon \left[ \left(\frac{\sigma}{r}\right)^{12} - \left(\frac{\sigma}{r}\right)^6 \right] \]

Test setup

  • Two atoms separated by a distance equal to \(\sigma\)
  • Parameters:
    • \(\epsilon = 1\)
    • \(\sigma = 1\)
    • \(r = \sigma\)

Expected result

At \(r = \sigma\), the Lennard-Jones potential is exactly zero:

\[ V(\sigma) = 0 \]

The test confirms that the implementation reproduces this property.

Test 3: Total energy "E"

The total energy is defined as:

\[ E = T + V \]

Test setup

  • Input values:
    • \(V = 10\)
    • \(T = 5\)

Expected result

\[ E = 10 + 5 = 15 \]

This test verifies that the E function performs a simple and correct summation without hidden scaling factors or side effects.

Verification criteria

Each test is evaluated using an absolute tolerance of:

\[ |E_{\text{computed}} - E_{\text{expected}}| < 10^{-6} \]

This tolerance is sufficient to detect implementation errors while remaining robust against floating-point rounding.

Design notes

  • Each function is tested independently, avoiding coupled failures.
  • All test cases use analytically exact reference values.
  • No stochastic elements or time integration are involved.
  • This test is suitable for:
    • Regression testing,
    • Validation after refactoring,
    • Cross-checking alternative implementations.
Note
These tests assume non-relativistic classical mechanics and neglect any long-range corrections, cutoffs, or unit conversions outside the energies module.
